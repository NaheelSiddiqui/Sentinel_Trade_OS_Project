# SentinelTrade Qt GUI

A clean, simple GUI for the multi-threaded stock exchange simulator. Reads live logs generated by the backend and displays them in real-time.

## Building

### Prerequisites
```bash
# Ubuntu/WSL
sudo apt-get install qt6-base-dev qt6-tools-dev cmake
```

### Build Steps
```bash
cd gui
mkdir build && cd build
cmake ..
make
```

### Run
```bash
# Make sure backend is running in another terminal
cd ..  # go back to SentinelTrade root
../backend/sentinel_trade &

# Then in another terminal, from the gui directory:
./build/sentinel_trade_gui
```

## Architecture

The GUI doesn't communicate with the backend while it's running. Instead, it **monitors log files** that the backend writes to:
- `trades.log` — all matched trades
- `system.log` — all system events

This approach is simple and doesn't require complex IPC or threading in the GUI.

### File Watcher
The `LogMonitor` class watches both log files for changes using Qt's `QFileSystemWatcher`. When a log is updated, it parses the new entries and emits signals to refresh the UI.

### Tabs

1. **Market** — Live ticker prices (symbol, price, % change)
2. **Order Book** — Bid/ask ladder for selected stock
3. **Trades** — Table of recent matched trades (shows buyer/seller trader IDs)
4. **System** — Statistics, trade progress bar, system log output

### Key Classes

- `LogMonitor` — watches and parses log files
- `MainWindow` — main window with tabs
- `MarketWidget` — ticker display
- `OrderBookWidget` — bid/ask tables
- `TradeLogWidget` — trade execution history
- `SystemWidget` — stats and system log

## Design Notes

**Why read logs instead of live IPC?**
- Simple — no shared memory or pipes to set up
- Non-invasive — backend code unchanged
- Safe — no threading issues in GUI
- Easy to debug — logs are human-readable files

**Refresh rate**
- UI updates every 500ms (controlled by QTimer in MainWindow)
- Log files are parsed and cached in memory for fast display
- No busy-waiting or excessive file I/O
